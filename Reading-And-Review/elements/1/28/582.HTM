通用解题路径


大道至简，开宗明义。通用解题路径就是学习、讨论、思考、实践。


2018


最初我的问题是，如何降低我们探索一个答案的成本。

因为答案很多，如果投入足够多的时间，迟早会发现，我们需要做的，主要是降低这个过程的成本。

于是有两种思路：

一是从直接探索（单干）到间接探索（借力）。

二是从行为探索（实战）到思维探索（模拟）。







结合这两种思路，2018年我形成一个矩阵，发现挺实用的。


  • 直接探索 + 行为探索 = 实践 = 试错来找答案，成本高，反馈快，依赖外部环境的支持。

  • 直接探索 + 思维探索 = 思考 = 做思维实验，成本较低，但反馈不够，依赖前期的积累。

  • 间接探索 + 思维探索 = 讨论 = 把想法给他人，在他人脑子里作思维实验，成本较低，反馈尚可。

  • 间接探索 + 行为探索 = 学习 = 通过阅读和观察等方式，直接获取他人的显性或隐性的知识和技能，为我所用，成本低，效果好。




2020


之后，我想到另一个角度，是从问题角度切入，这是2020年写《普罗米修斯的日课》过程中想到的。这个视角也是对之前理论的很好补充，因为原来的模式，只能告诉大家，寻找问题答案时，应该从成本出发，先学习、讨论，再思考和实践。现在有了问题矩阵，就可以更好理解这个解题过程。我们把问题分为四类：





第一种问题：已知的已知，有标准答案的问题，而且我们也知道谁掌握这个答案，所以最好的办法就是阅读这些老师的书籍，或者观察他们的行为，由此获得答案。这是最高效的方式。所以，如果问题是已知的已知，那么最好用学习来获得答案。


第二种问题：未知的已知，有标准答案的问题，但是我们不知道谁掌握这个答案，那么只好去提问、倾听和反馈，通过分享自己的思考，通过请教比自己有见识和阅历的人，来寻找这个掌握答案的人。最后你会通过拼接不同的答案，获得最终的答案。或者通过找到有用的关键词作为线索，最后找到想到的答案。


第三种问题：已知的未知，没有标准答案的问题，许多人也知道这个问题没有标准答案，所以都在研究中。此时，你可以去学习和讨论，但都只是为了收集更多基础素材，助力自己的思考。这个过程就像是做数学题目，你知道了公式，掌握了数据，你要做的是去计算，得出一个答案。


第四种问题：未知的未知，没有标准答案的问题，甚至许多人也不知道这个问题的存在，你可以在前期学习、讨论和思考，不断缩小探索的范围，但是最后你只能是到水里去，才能学会游泳。你必须通过实践来获得有效反馈，不断迭代优化自己的答案。


至此，我认为标准解题方式就可以成为一个基础模块了，可以拿来指导我们解决许多问题。而且这个方法，可以兼容前人的许多解题思路和建议。比如说：


  • 学习 + 思考 = 学而不思则罔，思而不学则殆。吾尝终日而思矣，不如须臾之所学。

  • 学习 + 实践 = 读万卷书，行万里路。

  • 学习 + 讨论 = 与君一席话，胜读十年书。

  • 思考 + 实践 = 凡事预则立，不预则废。实践是检验真理的唯一标准。



同时，我们发现，主题学习、深度工作等很多概念，只要是涉及解题的，最后可以还原为这个基础模块。

比如说，我现在如果遇到一个全新的问题，要去解决，我会开始组合学习、讨论、思考和实践。我们来马上生成一个操作清单：


  • 学习：快速找到知识源头的有效信息，先看百度词条了解大局，获得广度；然后找到感兴趣的点，找到学术论文阅读，了解更多理论，获得深度；再阅读一些前人的文章，来获得更多认识。如果是技能类的，则跟着专业人士观摩学习。

  • 讨论：把前面学习过程中发现的疑问梳理出来，请教该领域的专业人士，获得更多认识。

  • 思考：在写作过程中提炼观点，梳理思路，形成自己的认识。

  • 实践：用实际行动检验自己的理论，证明或者证伪，并根据反馈不断打磨和完善，最后形成操作清单，夯实自己的自主能力基础。


希望这个简单粗暴的小工具对大家有用。希望大家能举一反三。
